Redis Setup (Required for Celery):
Windows - Option 1 (Docker - Recommended):
    docker run -d -p 6379:6379 --name redis redis:latest
    docker start redis  (if already created)

Windows - Option 2 (WSL):
    wsl
    sudo apt-get update
    sudo apt-get install redis-server
    redis-server

Windows - Option 3 (Download):
    Download from: https://github.com/microsoftarchive/redis/releases
    Extract and run: redis-server.exe

Verify Redis is running:
    redis-cli ping  (should return PONG)
    Or: docker exec redis redis-cli ping

Start Celery (after Redis is running):
    Terminal 1: celery -A core worker --loglevel=info
    Terminal 2: celery -A core beat --loglevel=info

Production Setup (No Docker):
    On production server, run:
    sudo bash scripts/setup-celery-services.sh
    
    This will:
    - Install Redis as systemd service
    - Create Celery Worker systemd service
    - Create Celery Beat systemd service
    - Enable all services to start on boot
    
    See: docs/Working/Celery-Production-Setup.md

Local Development (No Celery):
    App works without Celery/Redis running!
    - Django server starts normally
    - API endpoints work (return empty data until Celery collects it)
    - Frontend works (shows "No data" until Celery collects it)
    
    To run monitoring checks manually:
    python manage.py run_monitoring_check
    
    See: docs/Working/Celery-Local-Development.md

PowerShell Venv Activation:
    Correct path (venv, not .venv):
    cd backend
    .\venv\Scripts\Activate.ps1
    
    If you get "The module '.venv' could not be loaded" error:
    
    Option 1 - Change execution policy (one-time):
    Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
    
    Option 2 - Use CMD instead:
    cmd
    cd backend
    venv\Scripts\activate.bat
    
    Option 3 - Use helper script:
    .\activate-venv.ps1
    
    See: docs/Working/PowerShell-Venv-Activation-Fix.md
